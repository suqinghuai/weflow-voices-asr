# WeFlow 语音转文字插件

这是一个适配 WeFlow 微信聊天记录导出软件 4.2.1 版本的插件，功能是批量对导出的 wav 格式原始音频文件使用 ASR 大模型进行语音转文字，并且将转文字结果填入替换原始聊天记录文件里面，再对聊天记录进行分割，方便转为 PDF。

## 功能特性

- **批量语音转文字**：自动识别并处理 `media/voices` 目录下的所有 WAV 音频文件
- **智能替换**：将语音转文字结果自动替换到聊天记录 HTML 文件中
- **聊天记录分割**：将长聊天记录分割成多个小文件，方便转为 PDF
- **双版本设计**：
  - 使用版：所有脚本都打包为了 exe，放入导出的聊天记录文件夹里面后按顺序运行即可
  - 开发版：提供完整的 Python 源代码，方便开发者进行二次开发

## 系统要求

- **使用版**：Windows 10 及以上操作系统
- **开发版**：
  - Python 3.7 及以上
  - 依赖库：`requests`

## 安装和使用（使用版）

1. **准备工作**：
   - 使用 WeFlow 4.2.1 版本导出微信聊天记录，确保导出了原始语音文件（wav 格式）
   - 导出的聊天记录文件夹中应包含 `media/voices` 目录，里面存放着 WAV 格式的语音文件

2. **使用步骤**：
   - 将安装包中所有文件复制到导出的聊天记录文件夹根目录
   - 按以下顺序运行：
     1. 运行 `1.转文字.exe`：批量处理语音文件，生成语音转文字结果
     2. 运行 `2.写入.exe`：将转文字结果替换到聊天记录 HTML 文件中
     3. 运行 `3.分割.exe`：将聊天记录分割成多个小文件，方便转为 PDF

3. **查看结果**：
   - 语音转文字结果会保存在 `语音转文字结果.jsonl` 文件中
   - 替换后的聊天记录会直接更新到原 HTML 文件中
   - 分割后的聊天记录会保存在 `分割结果` 文件夹中

## 开发版使用指南

1. **环境搭建**：
   - 克隆项目到本地
   - 安装依赖：`pip install requests`

2. **配置修改**：
   - 编辑 `config.ini` 文件，设置 API 相关参数

3. **运行步骤**：
   - 运行 `wav_to_txt.py`：批量处理语音文件，生成语音转文字结果
   - 运行 `write_html.py`：将转文字结果替换到聊天记录 HTML 文件中
   - 运行 `cut_html.py`：将聊天记录分割成多个小文件，方便转为 PDF

## 项目结构

```
├── wav_to_txt.py        # 语音转文字处理脚本
├── write_html.py        # 将转文字结果写入 HTML 文件脚本
├── cut_html.py          # 分割聊天记录脚本
├── config.ini           # 配置文件
├── build/               # 打包生成的文件
├── dist/                # 打包后的可执行文件
└── media/               # 示例媒体文件目录
    └── voices/          # 示例语音文件
```

## 配置说明

`config.ini` 文件包含以下配置项：

```ini
[API]
URL = https://api.siliconflow.cn  # ASR大模型 API 地址
KEY = sk-xxxxxxxxxxxxxxxxxxxxxxx  # API 密钥
MODEL = FunAudioLLM/SenseVoiceSmall  # 使用的 ASR 模型

[BASE]
result = 语音转文字结果.jsonl  # 语音转文字结果文件
cutmessages = 4000  # 分割聊天记录时每个文件的消息数量
```

## 工作流程

1. **语音转文字**：
   - 扫描 `media/voices` 目录下的所有 WAV 文件
   - 调用 ASR API 对每个文件进行语音转文字
   - 将结果保存到 `语音转文字结果.jsonl` 文件

2. **替换聊天记录**：
   - 加载语音转文字结果
   - 扫描当前目录下的所有 HTML 文件
   - 替换 HTML 文件中的 `[语音消息]` 为 `[语音转文字-xxx]`

3. **分割聊天记录**：
   - 读取 HTML 文件中的消息数据
   - 按配置的消息数量分割为多个文件
   - 将分割后的文件保存到 `分割结果` 文件夹

## 注意事项

1. **API 密钥**：
   - 请确保在 `config.ini` 文件中设置了正确的 API 密钥
   - 不同的 ASR 服务可能需要不同的 API 配置

2. **文件路径**：
   - 确保脚本或可执行文件与聊天记录 HTML 文件在同一目录
   - 确保 `media/voices` 目录存在且包含 WAV 格式的语音文件

3. **处理时间**：
   - 语音转文字处理时间取决于文件数量和大小
   - 请耐心等待处理完成

4. **分割大小**：
   - 可根据实际需要调整 `cutmessages` 参数，以获得合适的分割效果
